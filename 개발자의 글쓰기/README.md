### 1장 개발자가 알아야 할 글쓰기 기본
* 문장을 쉽게 쓰려면 핵심을 먼저 말하고 추가 설명이 필요한 경우 뒤에 붙여서 말하면 자연스러운 문단을 구성할 수 있다.
* 비즈니스 문서에는 위치랑 계층이 중요하다. 계층은 굻기, 모양, 밑줄, 줄, 줄 간 거리등으로 표현한다.
  
**문서 작성 시 서술식, 개조식, 도식의 활용**
| **서술식**                                                                                                                                                       |
| -----------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 사용처  | 설명, 개발 가이드 문서                                                                                                                                 | 
| 장단점  | 내용 설명력과 이해도는 높지만 작성 시간이 오래 걸리고 핵심을 파악하기 어렵다.    |

| **개조식**                                                                                                                                                       |
| -----------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 사용처  | 릴리스 문서, 장애 보고서                                                                                                                              | 
| 장단점  | 체계적이고 핵심 내용을 파악하기 쉽지만 디테일한 설명이 부족하다.    |

| **도식**                                                                                                                                                       |
| -----------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 사용처  | 표, 도식                                                                                                                              | 
| 장단점  | 시각적인 자료이기 때문에 자료, 구조를 이해하기 쉽지만 디테일한 설명이 부족하다.    |
